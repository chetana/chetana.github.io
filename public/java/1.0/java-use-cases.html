Fonctionnement d’une application entreprise Java

  • En général, une application d’entreprise est organisée de la façon suivante :

    • gestion de données dans une BDD (SQL, NoSQL)

    • implémente des processus métiers = tous les traitements qui vont être exécutés et nécessitent des données issues de la base (ex : création d’un compte bancaire…​) (applicatif Java)

    • interagit avec des utilisateurs ou d’autres applications via une couche de services (transiter les données de la couche métier vers l’IHM ou maj les données)

      • IHM divisée en 2 catégories :

        • Applications stand alone (legacy) : client lourd en Swing (directement déployé sur l’OS et intégré directement sur la couche de processus métiers)

        • Mobile ou Navigateur : via HTML, CSS et JS on va invoquer et exécuter des services écrits en Java pour obtenir ou maj des données

Domaines Techniques utilisés

  • JDBC : communication entre les BDD et les processus métiers, intéragir entre les BDD et l’applicatif Java (drivers natifs qui existent pour offrir une interface de communication vers les différentes sources de données)

  • Processus métiers : domaine roi de Java

  • JavaFX et Swing : applications legacy

  • Couche de Services : REST et SOAP

  • Applications mobiles : Android avec le langage Java et des APIs Java

  • Applets = code Java exécuté dans un navigateur (OBSOLETE au profit de HTLM5, CSS3 et JS)
