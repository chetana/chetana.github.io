Editions (Plateformes d’exécution)

Les éditions sont définies pour des cibles distinctes selon les besoins des applications.

JRE (Java Runtime Environment)

  • outils pour exécuter des applications Java

  • doit être installé seul sur les machines où des applications Java doivent être exécutées

JDK (Java Development Kit)

  • outils pour développer des applications Java

  • inclut le JRE

Editions de Java
Java SE (Standard Edition)

  • plutôt client (applications desktop)

  • fournit toutes les APIs de base pour développer des applications Java (compilateur, machine virtuelle, JDK)

  • sert de base aux autres plate-formes

Java EE (Enterprise Edition)

  • plutôt serveur

  • dédié au développement d’applications d’entreprise en fournissant des standards

  • s’appuie sur les APIs de base de SE (ex: Java SE a comme API JDBC et Java EE a des APIs de plus haut niveau comme JPA qui s’appuie sur JDBC pour améliorer la productivité (écrire de manière + rapide et + fiable les applications d’entreprise))

  • pendant de Java SE avec toutes les librairies qui manquent pour faire des applications d’entreprise (essentiellement celles qui permettent de gérer le web (HTTP, IHM web, webservices) et ceux pour l’accès aux BDDs (gestion de transaction, mapping objet-relationnel avec JPA))

  • ensemble d’API qui sont une surcouche de Java SE qui permet de faire des applications plus sophistiquées

JakartaEE

  • initialement, JavaEE était développé à travers le JCP

  • 2017 : Oracle donne JavaEE à la fondation Eclipse et pour des raisons de licence ⇒ JakartaEE

Java ME (Micro Edition)

  • environnement d’exécution et API pour le développement d’applications sur appareils mobiles et embarqués

Java Community Process (JCP)

  • les technologies Java évoluent au travers du JCP

  • organisation communautaire qui regroupe un certain nombres d’acteurs (individuels ou sociétés) pour définir les standards, des kits de compatibilité et des implémentations de référence

  • Oracle (depuis son rachat de Sun Microsystems) est le détenteur des marques déposées autour de la technologie Java et l’autorité suprême concernant les plates-formes Java

  • Des membres du JCP qui souhaitent enrichir la plate-forme Java doivent faire une proposition formalisée sous la forme d’une JSR

Java Specification Request (JSR)

  • suit un processus qui définit son cycle de vie autour de plusieurs étapes clés : drafts, review et approval.
