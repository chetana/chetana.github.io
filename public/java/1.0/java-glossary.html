Glossary

API

a group of class or interface definitions that give access to a service or a function

Buffer (mémoire-tampon)

zone en mémoire vive (ou du disque dur) utilisée pour stocker temporairement des données

Classpath

  • dans une app Java simple : on précise où sont nos libs, classes. S’il ne trouve pas une classe alors l’app regarde au niveau du système là où se trouve JRE (Rt.jar)

  • dans une app web : dans Tomcat on regarde le dossier WEB-INF/lib sinon server/lib sinon dans le JRE

Class

blueprint (plans) of the object

Comments

  • ignored by the computer

  • are added to the program to help humans describe something

Console

gestion du dispositif d’entrée/sortie intéractif lié à la JVM

Désérialisation

transformer un objet non décrit en Java en un objet Java réel et concret

Expression

a construct that evaluates to a single value

Instance

object instance (created with new keyword)

Introspection

examiner le contenu d’un objet sans connaître à priori le nom de sa classe

JAR (Java ARchive)

  • extension propre aux archives Java

  • ce type de fichier contient tout ce dont la JVM a besoin pour lancer un programme

Magic Number

  • définit le type de fichiers (4 octects)

  • "cafebabe" en Java

Main method

entry point of your Java application

Method signature

name + parameters list

Reference

  • contient un emplacement où l’objet est stocké dans la mémoire

  • pointeur, pointe vers l’objet ⇒ permet de manipuler l’objet

  • Un Handle est un type qui contient une pseudo adresse mémoire et qui autorise l’accès à la valeur sous-jacente.

Reflection

mécanisme qui permet de créer une classe sans passer par des méthodes explicites

Serialization

écrire des objets ou des graphes d’objets directement vers des flux, que ce soit des fichiers ou en mémoire

SerialVersionUID

hashcode calculé avec le nom et la structure de la classe

Statement

a complete command to be executed and can include one or more expressions

String literal

any sequence of characters surrounded by double quotes

Unicode

  • international encoding standard by which each letter, digit or symbol is assigned to a unique numerical value that applies accross different platforms and programs

  • ex: 'D'='\u0044'

Variables

way to store information on our computer the computer does the hard work of figuring out where they get stored on the RAM

== OpenJDK VS Oracle JDK

https://blogs.oracle.com/javamagazine/text-blocks-come-to-java https://marxsoftware.blogspot.com/2019/10/better-npe-messages-in-jdk-14.html [Baeldung](https://www.baeldung.com/oracle-jdk-vs-openjdk)

OpenJDK (2007)

  • free and open-source implementation of the Java SE Platform Edition

  • official reference implementation of a Java SE since version 7

Differences :

1) Release Schedule (Oracle JDK VS OpenJDK) * every 3 years VS every 6 months * provides LTS for its releases VS supports the changes to a release only until the next version is released 2) Licenses * Oracle Binary Code License Agreement VS GNU General Public License (GNU GPL) version 2 with a linking exception ⇒ not for commercial use from a certain date VS free 3) Performances * No real technical difference since the build process for the Oracle JDK is based on that of OpenJDK * much better regarding resposiveness and JVM perf 4) Features * Flight Recorder, Java Mission Control, Application Class-Data Sharing VS Font Renderer * more Garbage Collection options and better renderers 5) Development and Popularity * Oracle Corporation + top-notch companies (RedHat, Azul Systems, IBM, Apple Inc., …​) (used in Android Studio or IntelliJ IDEA) VS Oracle, OpenJDK and the Java Community (used in Linux distributions)

Le code source du JDK est passe de Mercurial à Github : Raisons : - performances de Github (rapidite et fiabilite = downtime) - outils de gestion du jdk (tracages de versions, de bugs, de nouvelles features) n ont pas ete migrees vers Github mais continue a utiliser leur propre workflow ⇒ on migre avec un certain nombre de precautions
