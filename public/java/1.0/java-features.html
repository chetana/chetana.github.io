Caractéristiques

Java est interprété

  • Après compilation du code source d’une application Java (.java), on obtient un binaire portable = bytecode (.class) puis exécuté par un interpréteur Java : la Java Virtual Machine (JVM).

Java est portable, indépendant de la machine et de l’OS

  • Write Once Run Anywhere (WORA) : le bytecode ne contient pas de code spécifique à une plateforme particulière, il suffit juste d’avoir une JVM sur sa machine.

Java est un langage de programmation orienté objet

  • Chaque fichier source contient la définition d’une ou plusieurs classes utilisées pour former une application.

  • Pas complètement objet car Java définit des types primitifs.

Java est simple

  • Abandon de notions mal comprises ou mal exploitées des autres langages (pointeurs, héritage multiple, surchage des opérateurs…​)

Java est fortement typé

  • Toutes les variables sont typées.

Java assure la gestion de la mémoire

  • Allocation de la mémoire automatique.

  • Récupération automatiqe de la mémoire inutilisée grâce au Garbage Collector (GC) ⇒ robuste.

Java est sûr

  • Le code se trouve à l’intérieur d’une sandbox JVM.

Java est économe

  • le pseudo code a une taille relativement petite car les bibliothèques requises ne sont liées qu’à l’exécution.

Définitions

JVM (Java Virtual Machine)

  • process dédié sur chaque OS

  • lire et charger les fichiers .class de notre code afin de l’exécuter

  • s’appuie sur un ensemble de libs standards (qui sont aussi des .class)
