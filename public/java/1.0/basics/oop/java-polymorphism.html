Polymorphisme

Le polymorphisme est la capacité, pour un même message, de correspondre à plusieurs formes de traitements selon l’objet auquel ce message est adressé. La gestion du polymorphisme est assurée par la machine virtuelle dynamiquement à l’exécution.

Le transtypage induit par l’héritage facilite le polymorphisme

L’héritage définit un cast implicite de la classe fille vers la classe mère : on peut affecter à une référence d’une classe n’importe quel objet d’une de ses sous-classes.

// Exemple : la classe Employe hérite de la classe Personne
    Personne p = new Personne ("Dupond", "Jean");
    Employe e = new Employe("Durand", "Julien", 10000);
    p = e ; // ok : Employe est une sous-classe de Personne
    Objet obj;
    obj = e ; // ok : Employe hérite de Personne qui elle même hérite de Object

Une interface sert à définir un supertype

  • L’implémentation d’une interface définit un cast implicite de la classe fille vers cette interface.

Façons d’accéder à un objet

  • A Java object may be accessed with :

    • a reference of the same type ;

    • a reference that is a superclass ;

    • a reference that defines an interface the object implements.

The object type determines which properties exist within the object in memory.
The reference type to the object determines which methods and variables are accessible to the Java program.

Virtual method

Method in which the specific implementation is not determined until runtime

// Lorsqu'un objet a été affecté à une référence d'un autre type, seuls les membres accessibles au type de cette nouvelle référence sont disponibles.
// Par exemple, dans le code suivant, on ne peut plus accéder aux propriétés de la classe Lemur avec l'objet lemurAsObject :

public class Lemur {
    public String name = "titi";
    public static void main(String[] args) {
        Lemur lemur = new Lemur();
        System.out.println(lemur.name); // titi
        Object lemurAsObject = lemur;
        System.out.println(lemurAsObject.name); // DOES NOT COMPILE
    }
}

// Si on appelle une méthode virtuelle sur un objet qui redéfinit une méthode, la méthode redéfinie est appelée même si l'appel est fait sur une référence de type de la classe parent ou si l'appel est fait au sein de la classe parent :
public class Bird {
    public String getName() {
        return "Unknown";
    }
    public void displayInformation() {
        System.out.println("The bird name is : " + getName());
    }
}

public class Peacock extends Bird {
    public String getName() {
        return "Peacock";
    }

    public static void main(String[] args) {
        Bird bird = new Peacock();
        bird.displayInformation(); // The bird name is : Peacock
    }
}
