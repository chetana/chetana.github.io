Classe, Variables et Méthodes

Variables

Declaration / Initialization

double d1, double d2; // KO
int i1; int i2; // OK
$ or _ // OK

Les variables définies dans une boucle ou une instruction conditionnelle ne peuvent pas être accédées en dehors du scope dans lequel elles sont créées.

Table 1. Types de variables
Type de variable Description Portée

Local variable

must be initialized before use

Les variables locales sont accessibles de leur déclaration à la fin du bloc.

Instance variable

  • belong to an instance of a class and represent the state of an instance

  • every instance has its own copy of an instance variable

  • every instance can have a different value

  • no need to be initialized before use ⇒ default values provided

Les variables d’instance sont accessibles de leur déclaration jusqu’à ce que l’objet soit garbage collected.

Static variable

  • declared using a static modifier

  • known as static member variables

  • every instance of that class shares the same static variable

Les variables de classe sont accessibles de leur déclaration jusqu’à ce que le programme termine.

Méthodes

Instance methods

  • belong to an instance of a class

  • to use an instance method, we have to instantiate the class first

  • can access instance methods and instance variables directly

  • can access static methods and static variables directly

Static methods

  • declared using a static modifier

  • can’t access instance methods and instance variables directly

  • used for operations that don’t require any data from an instance of the class

  • we can’t use the this keyword

  • whenever you see a method that does not use instance variables ⇒ should be declared as a static method

Ordre d’initialisation dans une classe

  1. S’il y a une classe parent, elle est initialisée en premier

  2. Les variables et initialiseurs static dans l’ordre dans lequel ils apparaissent dans le fichier

  3. Les variables et initialiseurs d’instance dans l’ordre dans lequel ils apparaissent dans le fichier
