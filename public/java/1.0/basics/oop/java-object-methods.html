Object Methods

public String toString()

Renvoie le type de l’objet suivi d’un code d’identification unique pour identifier l’instance dans la JVM

public boolean equals(Object object)

Il faut redéfinir cette méthode si l’on veut vérifier que deux instances DISTINCTES sont fonctionnellement équivalentes. Il est généralement nécessaire de redéfinir hashcode avec equals().

Contrat à respecter : * Reflexive : for every x different than null, x.equals(x) should return true * Symmetric : for every x and y different than null, x.equals(y) should return true if y.equals(x) returns true * Transitive : for every x, y and z different than null, if x.equals(y) returns true and y.equals(z) returns true, then x.equals(z) should return true * Consistent (l’appel à cette méthode doit toujours retourner le même résultat si on ne change pas les objets) : for every x and y different than null, multiple invocations of x.equals(y) consistently return true or false for every x different than null, x.equals(null) should return false

public int hashCode()

  • number that puts instances of a class into a finite number of categories

  • valeur entière représentant la réduction d’un objet

  • whenever you override equals(), you are expected to override hashCode() !

    • Si deux objets sont égaux au sens d’equals() alors ils doivent avoir le même hashcode

    • En revanche, deux objets non égaux peuvent avoir le même hashCode (collision)

  • Within the same program, the result of hashCode() must not change ⇒ you should not include variables that change

  • if equals() returns true when called with 2 objects, calling hashCode() on each of those 2 objects must return the same result

  • if equals() returns false when called with 2 objects, calling hashCode() on each of those 2 objects does not have return a different result

  • cannot use more variables than equals()

  • improper hashCode does not mean code won’t compile

  • exemple : dans un jeu de cartes, le hashCode renvoie la couleur de la carte. Ce hashCode est calculé sur tout ou partie des variables utilisées dans equals().

Utilité ?

  • HashCode est ce qui est regardé en premier dans les Set ⇒ + facile et + rapide de comparer les hash

Exemple 1 : - on crée deux objets Employee avec le même id - new HashMap<Employee, String>() - on les met dans une hashmap map.put(emp1, "Jack") et map.put(emp2, "Jack") - map.size() ? ⇒ 2

  • de même avec new HashMap<Integer, String>()

  • map.size() ? ⇒ 1 car Integer redéfinit les méthodes hashCode() et equals()
