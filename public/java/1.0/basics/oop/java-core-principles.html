Programmation Orientée Objet

L’idée de base de la programmation orientée objet est de rassembler dans une même entité appelée objet les données et les traitements qui s’y appliquent.

Class, Object/Instance, Reference

  • Analogie avec la construction d’une maison

    • Class : plans pour la maison

    • Object/Instance : chaque maison qu’on crée avec le mot-clé new

    • Reference : chaque maison a une adresse que l’on peut écrire sur un bout de papier

      • on peut copier autant de fois une référence mais elles désigneront toujours le même objet (plusieurs papiers avec l’adresse de la même maison)

      • on peut les passer comme paramètres aux méthodes

File Structure

  • PIC : Package Imports Class = order declaration required then fields and methods

  • At most one public class (has to match the file name)

  • Comments (ignored)

  • Imports

    • java.lang automatically imported

    • wildcards : * only match class names

    • naming conflicts
