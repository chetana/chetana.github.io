Overriding

  • Define a method in a child class that already exists in the parent class with same signature (same name, same arguments)

  • Run Time Polymorphism / Dynamic Method Dispatch : the method that is going to be called is decided at runtime by the JVM

  • Can be overriden :

    • instance methods

    • inherited methods

  • Cannot be overridden :

    • variables

    • static methods

    • constructors and private methods

    • final methods

Rules

  • it must have the same signature (same name and same arguments)

  • it can’t have a lower access modifier = child method must be at least as accessible or more than the method in the parent class

  • it must not throw a new or broader checked exception than the parent method

  • covariant return types : return type must be the same or can be a subclass of the return type in the parent class
