Fiabilité d’une application Java

La fiabilité d’une application Java se base sur : * la correction * la robustesse

Correction

Le fait que le logiciel donne des résultats corrects

Assert

  • Expression booléenne où l’on souhaite vérifier si quelque chose est vrai

  • Syntaxe : _assert expr1 [: expr2] (avec expr1 une expression booléenne et expr2 un type quelconque)

  • Exécution :

    • expr1 est évaluée

    • si true → le code continue son exécution

    • si false → une AssertionError est lancée

    • expr2 must return a value : convertie en chaîne, expr2 es utilisée comme message

  • Une assertion ne doit pas altérer la variable qu’elle utilise

java -ea <className> // activate assertions

Robustesse

Capacité du logiciel à fonctionner, même en présence d’évènements exceptionnels = exceptions

cf java-exceptions.adoc et java-try-catch.adoc
