Conversions - Cast pour les objets

Fait de forcer le compilateur à considérer un objet comme étant d’un type qui n’est pas le type déclaré ou réel de l’objet

Table 1. Cast Types
Type Description

Unrelated types cast

Impossible

Upcast

* un objet est considéré comme une instance d’une des classes ancêtres de sa classe réelle * toujours possible avec la relation est-un de l’héritage : tout objet peut être considéré comme une instance d’une classe ancêtre * souvent implicite

Downcast

* un objet est considéré comme une instance d’une sous-classe de sa classe de déclaration * pas toujours possible (vérification à l’exécution) * doit toujours être explicite ! * ex: Grande extends Petite Grande x = new Grande(); Petite y = x; Grande z=(Grande)y;
