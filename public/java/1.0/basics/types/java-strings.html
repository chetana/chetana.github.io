Strings

Les Strings sont des chaînes de caractères.

Un objet String est immutable

  • Non modifiables après création

  • Si on assigne une nouvelle valeur, on crée un nouvel objet ⇒ coûte cher en terme de performance

    • ex: lastString = lastString + 50; // on crée un nouvel objet String

String Pool

  • Zone mémoire contenant tous les String littéraux (ex: "myString") pour optimiser la mémoire

  • les String sont stockés dans le pool au compile-time

  • Ceux qui ne sont pas dans le pool sont garbage collected comme tout autre objet

Méthodes

  • equals()

  • equalsIgnoreCase()

  • charAt()

  • subString() : n’inclut pas endIndex

  • indexOf()

  • startsWith()

  • endsWith()

  • trim() : enlève espaces et tabulations au début et à la fin

  • replace()

  • concat()

  • toUpperCase()/toLowerCase()

  • split()

Equals VS reference equality (==)

String x = "Hello";
String y = "Hello";
System.out.print(x == y); // true car les littéraux sont dans le pool (même objet)

String x1 = "Hello";
String y1 = " Hello".trim();
System.out.print(x1 == y1); // false : les littéraux sont stockés dans le pool au compile-time et y1 est traité au run-time

StringBuffer

  • Chaînes mutables

  • Opérations plus rapides (append, insert, deleteCharAt, deleve, reverse)

  • Thread-safe : toutes les méthodes sont synchronisées ⇒ ralentit le programme

StringBuilder

  • même chose que StringBuffer mais non thread-safe ⇒ plus rapide que StringBuffer

  • size : number of characters currently in the sequence

  • capacity : number of characters than the sequence can hold (16 by default)
