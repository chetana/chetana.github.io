Overloading

  • Compile Time Polymorphism : the compiler decides which method is going to be called based on method name, return type and argument list

  • Very handy, it reduces duplicated code and we don’t have to remember multiple method names

  • Can be overload :

    • instance methods

    • static methods

Rules

  1. Methods must have the same name

  2. Methods must have differents parameters

  3. May have different return types, access modifiers, throw different checked and unchecked exceptions

Choix de la méthode parmi plusieurs existantes

  • Java choisira la méthode la plus spécifique selon les règles suivantes (exemple avec la méthode glide(1,2))

    • Correspondance des types : public void glide(int i, int j) {}

    • Types primitifs plus larges : public void glide(long i, long j) {}

    • Type autoboxed : public void glide(Integer i, Integer j) {}

    • Varargs : public void glide(int…​ nums) {}
