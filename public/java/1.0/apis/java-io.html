Java I/O (Input/Output)

Mécanismes qui permettent l’échange des données d’une application Java avec l’extérieur.

L’extérieur peut désigner :

  1. le disque

  2. le réseau

    • Web (HTTP) : API Servlet qui s’appuie beaucoup sur Java I/O (utilisé par REST)

    • BDD

  3. la mémoire off heap

    • accessible au travers d’extensions de Java I/O

    • permet d’étendre les capacités mémoires (teraoctet)

java.nio.Path (interface) (Java 7)

Ce sont des wrappers de chemins ⇒ aucun fichier n’est créé (aucun accès disque n’est fait) !

java.nio.Path

  • supporte les symbolic links

  • java.net.URI : Uniform Resource Identifier

    • correspond à une String pour caractériser une ressource en commençant par une chaîne de caractère pour définir le type de ressource tel que file://, http://, https:// ou ftp://

    • n’accepte que des paths absolus

Path path1 = Paths.get("pandas/cuddly.png"); Path path2 = Paths.get("/", "home", "zoodirector"); Path path1 = FileSystems.getDefault().getPath("pandas/cuddly.png");

Path path1 = Paths.get(new URI("file://pandas/cuddly.png")); // Lève une exception au Runtime : URI doivent être absolues

Path path2 = Paths.get(new URI("http://www.wiley.com")); URI uri = path2.toUri();

FileSystem fileSystem = FileSystems.getFileSystem(uri); Path path3 = fileSystem.getPath("duck.txt");

Nouvelles méthodes Stream

2 stratégies de recherche Breadth-first : on part du dossier Root et on avance niveau par niveau en regardant chaque élément Depth-first : on part du dossier Root jusqu’à une branche arbitraire et on remonte vers root (utilisé par les Streams par défaut avec une profondeur de recherche de Integer.MAX_VALUE walk(Path) : permet de parcourir un dossier find(Path, int, BiPredicate) : permet de chercher un fichier qui respecte le BiPredicate et on doit indiquer la profondeur de recherche list(Path) : permet de lister le contenu d’un dossier lines(Path) : permet de lire le contenu d’un fichier et qui est un équivalent de Files.readAllLines() mais qui évite d’avoir un OutOfMemoryError si le fichier est trop large car avec les Streams, on ne lit qu’une partie de notre fichier à la fois

Path path = Paths.get("/bigcats"); try { Files.walk(path) .filter(p → p.toString().endsWith(".java")) .forEach(System.out::println); } catch (IOException e) { // …​ }

Attention: Si vous utilisez les symbolic links (off par défaut), faites attention aux circular paths, c’est-à-dire rester bloqué dans une loop infinie vu que le symbolic link nous ramène dans un dossier où il est par exemple et le programme vous lèvera une FileSystemLoopException.

Stream<Path> stream = Files.find(path, 10, (p, a) → p.toString().endsWith(".java") && a.lastModifiedTime().toMillis() > 1420070400000l); stream.forEach(System.out::println); // a : BasicFileAttributes
