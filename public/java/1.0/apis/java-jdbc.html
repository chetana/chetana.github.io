Java DataBase Connectivity (JDBC)

Classes Java permettant de se connecter et d’interagir avec des BDDs.

URL Protocol : Product/VendorName:DBConnectionDetails jdbc:[vendorName]://[host][:port][database][?property1][=value1]

Il y a 4 interfaces principales faisant partie du JDK.

Table 1. Composants du JDK pour accès à la base de données
Nom Description Exemple

Driver

Sait comment établir une connexion à la BDD

DriverManager ou DataSource

Connection

Sait comment communiquer avec la BDD

DriverManager.getConnection() → SQLException (getMessage(), getErrorCode(), getSQLState())

Statement

Stocke et exécute une requête SQL

  • connection.createStatement(ResultSet.TYPE_FORWARD_ONLY, ResultSet.CONCUR_READ_ONLY) // by default

  • boolean execute()

  • ResultSet executeQuery()

  • int executeUpdate()

PreparedStatement

Précompile la requête et permet d’utiliser un système de requête à trous

ResultSet

Stocke les lignes résultant de l’exécution de la requête SQL

rs.next() // permet de parcourir le RS en utilisant le cursor qui pointe sur la position courante des données
Ne pas oublier de fermer les ressource ! * Connection closed ⇒ Statement and ResultSet also closed * Statement closed ⇒ ResultSet also * if Statement reused ⇒ closes the previous ResultSet 
import java.sql.*;

public class MyFirstDatabaseConnection { public static void main(String[] args) throws SQLException { String url = "jdbc:derby:zoo"; try (Connection conn = DriverManager.getConnection(url); Statement st = conn.createStatement(); ResultSet rs = st.executeQuery("select name from animal;");) { while (rs.next()) { System.out.println(rs.getString(1)); } } } }

ResultSetMetaData et DataBaseMetaData

Donnent accès à des informations globales sur une requête ou une BDD
