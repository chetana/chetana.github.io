Singleton

Pouvoir créer un objet en mémoire une seule fois et qui est partagé par plusieurs classes dans notre application. (1 classe = 1 instance) * Intérêts : économiser de la mémoire (un objet consomme de la mémoire, si une seule instance me suffit, pas besoin d’en créer plus) économiser un peu de CPU : plus je crée des objets, plus j’ai des chances de créer de la pression sur le GC qui va regarder si les objets sont non utilisés et les jeter * Use cases : marche particulièrement bien pour les classes de service (une classe qui gère la connexion à MySQL) manage a reusable cache of objects manage write access to a log file provide central access to app config data

Première implémentation du Pattern Singleton

public class MySQLDBService { // contrôler qui peut construire cette classe // interdire tout le reste de mon application d’instancier cette classe private MySQLDBService() { …​ }

// je crée une instance à l'intérieur de cette classe
// pour la rendre disponible au travers de la classe avant que quelqu'un ait une référence sur la classe => accesseur static
static MySQLDBService service = new MySQLDBService();
  // disponible à l'extérieur
  static MySQLDBService getInstance() {
    return service;
  }
}
pas la facon que les gens preferent car l’instanciation de mon objet service n’est pas controlee. On aimerait une instanciation lazy, cad que le service soit instancie au moment ou on en a besoin (appel de getInstance()). Si la classe est très simple et n’a pas d’autre membre statique, cet initialiseur statique va etre invoque sur un appel à getInstance() Sinon s’il y a des membres statiques ou des references ou que cete classe est referencee par autre chose ⇒ je vais payer le prix de la creation d cette instance sans appel de getInstance() ⇒ risque de creer un objet dans l’appli inutiise

Règles: attributs private (instance) constructeur private -→ marque la classe final implicitement On instancie le singleton lorsque l’on crée notre classe avec notre variable static et final instance. public class HayStorage { private int quantity = 0; private hayStorage() {}

private static final HayStorage instance = new HayStorage();
public static HayStorage getInstance() {
  return instance;
}
    public synchronized void addHay(int amount) {
      quantity += amount;
    }
    ....
}

Un autre façon consiste à utiliser la Lazy instantiation en créant notre objet dans la méthode getInstance(). Ainsi, notre singleton n’est crée qu’au moment où il est appelé la première fois.

public class HayStorage { private int quantity = 0; private hayStorage() {}

private static final HayStorage instance;
public static synchronized HayStorage getInstance() { // synchronized pour être thread safe
  if (instance == null) {
      instance = new HayStorage();
  }
  return instance;
}
    public synchronized void addHay(int amount) {
      quantity += amount;
    }
    ....
}
