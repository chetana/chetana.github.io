API Reflection

API bas niveau qui manipule les objets

  • espèce d’ouvre-boîte qui permet d’ouvrir les objets, de regarder ce qu il y a dedans sans nécessairement connaître à la compilation le nom de la classe ni le nom des méthodes et des champs

  • permet de faire de l’introspection = examiner le contenu d’un objet sans connaître à priori le nom de sa classe

à priori signifie à la compilation.

  • Utilisé au coeur du fonctionnement de tout un tas de frameworks dans l’écosystème Java

    • eg : Jakarta EE

    • eg : Hibernate

    • eg : Spring

    • tous les frameworks qui font des ihm web

    • tous les services REST

Class

  • Point d’entrée principal de l’API Reflection

  • Modélise les classes en Java

Table 1. Façons d’obtenir une instance de Class
Moyen Exemple

à partir d’un objet

Object o = …​ Class classe = o.getClass() // retourne la classe d’implémentation de o ⇒ je peux récupérer la référence de l’objet quand bien même je ne connais pas le type de l’objet à la compilation

à partir d’une référence

Class c = String.class; // retourne un pointeur vers l’instance de la class Class qui modélise la classe String

  • à partir de la méthode factory Class.forName(String str)

  • à la base de l’API Reflection : je donne un nom de classe à l’exécution de mon application et mon application est capable d’aller chercher cette classe, de la charger et d’avoir des informations dessus

Class c = Class.forName("java.lang.String");

Ces 3 façons d’obtenir une instance de Class qui modélise une classe particulière respectent le pattern singleton. NOTE: Une instance de Class n’existe qu’en un seul exemplaire dans la mémoire c1 == c2 <⇒ c1.equals(c2)

Instancier une classe par reflection

Class c = Class.forName("model.User"); // classe User dans le package model User user = (User) c.newInstance(); // permet d’instancier une classe sans le mot clé new et va appeler le constructeur vide de la classe (importance des beans en Java). De plus la méthode est dépréciée car il y a un nombre de vérifications de sécurité non faites

Récupérer les champs de la classe

Field : classe qui modélise le champ d’une classe Field f1 = c.getField("name"); // retourne le champ de c s’il existe sinon va explorer les superclasses Field[] f1Array = c.getFields(); // retourne les champs publics de c et ceux de ses superclasses

Field f2 c.getDeclaredField("name") // ici name est un champ private et pas d’exploration de la superclasse Field[] f2Array = c.getDeclaredFields(); // retourne l’intégralité des champs de la classe sans les superclasses Méthodes : f2.getName() Class f2.getType() Field fieldFromUser1 = f2.get(u1); // "Pierre" Field fieldFromUser1 = f2.get(u2); // "Paul"

f.set(u3, "Kylian");

on essaye de lire et fixer la valeur d’un champ private de l’extérieur de sa classe ⇒ Exception !

f.setAccessible(true); // porte dérobée qui demande à la JVM de fermer les yeux sur le fait que je demande la valeur d’u n champ private

il est possible d’activer des règles de sécurité au niveau de la JVM pour empêcher ce genre de méthode de fonctionner. L’invocation de setAccesible() est contrôlée par un security manager.

Récupérer les méthodes de la classe

Class c = Class.forName("model.User"); Method getMethod(String methodName, List<T>) // méthode publique Method[] getMethods() // retourne les méthodes publiques de la classe et des superclasses et les méthodes par défaut des interfaces implémentées par c Method getDeclaredMethod(String methodName, List<T>) Method getDeclaredMethods() // retourne l’intégralité des méthodes de la classe

getMethod("getName"); // simple getter, ne prend pas de paramètre getMethod("setName", String.class); // setter qui sette un nom avec un String

il est important de préciser le type du ou des paramètres car on peut avoir de l’overloading ! getMethod("setAmountSalary", int.class); // pour l’augmentation de salaire en nombre getMethod("setAmountSalary", float.class); // pour l’augmentation de salaire en pourcentage

Invoquer une méthode par reflection

Method getName = c.getMethod("getName"); getName.invoke(u1); Method setName = c.getMethod("setName", String.class); setName.invoke(u2, "Paul");

Accéder aux constructeurs d’une classe

Constructor getConstructor() Constructor[] getConstructors() // retourne les constructeurs publiques Constructor getDeclaredConstructor() Constructor[] getDeclaredConstructors() // // retourne les constructeurs publiques, privés, package-proctected et protected WARNING: on ne va pas explorer les superclasses car à partir d’un objet Constructor je peux instancier une classe et si je prends le Constructor d’un User, je m’attends à instancier un User. Si on avait pris les superclasses, on aurait instancier un objet qui n’est pas de type User…​

Constructor c1 = getConstructor() // constructeur vide Constructor c2 = getConstructor(String.class, int.class) // constructeur avec 2 paramètres

Invoquer un constructeur

Constructor c1 = getConstructor() // constructeur vide User user = c1.newInstance() NOTE: ce pattern est préféré au class.newInstance() car on a un checking de sécurité supplémentaire qui est fait : est-ce qu’on a bien accès à ce constructeur vide ou non ?

Constructor c2 = getConstructor(String.class, int.class) // constructeur avec 2 paramètres User user = c2.newInstance("Sophie", 34);

Modificateur d’un élément d’une classe

  • Concerne Field, Method, Constructor Modificateurs possibles :

  • visibilité

  • static/non static

  • final

  • transient

Modifier m = int field.getModifiers(); // cet entier est un champ de bits (l’information est portée par le fait que les bits sont à 0 ou 1) Modifier.isPrivate(m); // je passe cet entier à des méthodes factory
